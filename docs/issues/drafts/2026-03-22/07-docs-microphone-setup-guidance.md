# [DOCS] Add microphone setup guidance for reliable measurements

## Summary

Users repeatedly report that Tg works well only when microphone choice, placement, and environment are handled correctly, but the current documentation does not explain this clearly enough.

## Priority

- P2 Medium

## Labels

- `documentation`
- `usability`

## Why It Matters

External feedback suggests that microphone setup is one of the biggest determinants of whether users get useful results or conclude the software is inaccurate.

## Evidence

- Reddit and forum discussions repeatedly emphasize microphone quality and positioning.
- Earbud or external microphones are often reported to work better than laptop built-ins.
- Users value repeatable results but often need practical guidance to get them.

## Proposed Fix

Add a practical user-facing guide covering:

- microphone types that tend to work well
- placement advice
- quiet-environment recommendations
- common causes of noisy or unstable readings

## Acceptance Criteria

- [ ] The documentation includes practical microphone setup guidance.
- [ ] The guide explains common failure modes in plain language.
- [ ] The README or overview points users to that guidance.
