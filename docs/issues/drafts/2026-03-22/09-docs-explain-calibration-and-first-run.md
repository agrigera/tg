# [DOCS] Explain calibration and first-run setup in plain language

## Summary

External discussion shows that new users often do not understand why Tg needs calibration or how to know whether their first setup is trustworthy.

## Priority

- P2 Medium

## Labels

- `documentation`
- `onboarding`

## Why It Matters

The software asks users to do technically reasonable things, but the current documentation does not explain the reasoning clearly enough for first-time users.

## Evidence

- External users ask why calibration is needed at all.
- Community guides spend significant space explaining calibration and repeatability.
- Poor onboarding increases the chance that users misread bad setup as bad software.

## Proposed Fix

Add a plain-language first-run guide covering:

- why calibration exists
- what a good reference source looks like
- what to verify before trusting a measurement
- what to do when readings look implausible

## Acceptance Criteria

- [ ] Documentation explains calibration in non-specialist language.
- [ ] The first-run path includes a clear measurement sanity checklist.
- [ ] New users can identify the most common setup mistakes without reading forum threads.
